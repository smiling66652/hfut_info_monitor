## 报告漏洞

如果您发现安全漏洞，请**不要**通过公开 Issue 报告。

请通过电子邮件报告给项目维护者：

**📧 2240678683@qq.com**

### 报告内容

请包含以下信息：

- **漏洞类型**（例如：SQL 注入、XSS、权限提升等）
- **受影响文件/模块**
- **重现步骤**
- **潜在影响**
- **建议修复方案**（可选）

### 响应时间**

- **确认收到**：48 小时内
- **初步评估**：1 周内
- **修复发布**：评估后 2 周内（根据严重程度调整）

### 披露政策**

- 我们遵循**协调披露**原则
- 在漏洞修复之前，请**不要**公开披露
- 修复后，我们将：
  - 发布安全公告
  - 致谢发现者（如果您同意）
  - 发布补丁版本

## 安全最佳实践

### 对于使用者

1. **保护配置文件**
   - 不要将 `config/config.json` 提交到公开仓库
   - 使用环境变量或 `.gitignore` 排除敏感文件

2. **邮箱授权码**
   - 使用专用授权码，不要使用登录密码
   - 定期更换授权码

3. **Ollama 配置**
   - 不要将 Ollama 暴露在公网
   - 使用防火墙限制访问

4. **定期更新**
   - 保持依赖库更新
   - 关注安全公告

### 对于贡献者

1. **不要提交敏感信息**
   - 密码、Token、API Key
   - 内部服务器地址
   - 用户数据

2. **依赖安全**
   - 使用 `pip-audit` 检查依赖漏洞
   - 优先使用官方源

3. **代码审查**
   - 所有 PR 必须经过审查
   - 关注安全相关问题
